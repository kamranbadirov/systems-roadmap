# systems-roadmap

A personal learning roadmap through the deepest parts of Computer Science — Compilers, Programming Language Theory, Operating Systems, and Runtimes.

## What This Is

This is not a tutorial or a course. It is a structured, project-driven path for someone who already has a CS background but wants the foundational layers to *actually click* — not just be memorized for an exam.

The domain is **Programming Languages & Systems**: the subfield where people build languages, compilers, virtual machines, operating systems, and runtimes. LLVM, V8, the Dart VM, the Rust type system — this is that territory.

## The Roadmap

| Phase | Topic | Duration |
|-------|-------|----------|
| 0 | How Computers Work | 4–6 weeks |
| 1 | Operating Systems | 6–8 weeks |
| 2 | C & Systems Programming | 4–6 weeks |
| 3 ★ | Compilers | 10–14 weeks |
| 4 | Programming Language Theory | 8–10 weeks |
| 5 | Runtimes & Ecosystem | 6–8 weeks |

## Core Philosophy

1. Nothing moves forward until the current phase is solid.
2. Every concept gets a project.
3. Going deep first feels slower. It isn't.
